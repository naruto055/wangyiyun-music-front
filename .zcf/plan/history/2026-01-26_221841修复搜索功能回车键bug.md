## 🔍 问题分析

### 发现的问题

**1. 表层问题：按回车不搜索**
- 位置：[src/views/MusicList.vue:26](src/views/MusicList.vue#L26)
- 原因：缺少回车键事件监听
- 影响：用户无法通过回车键触发搜索

**2. 根本问题：Input组件v-model未实现** ⭐ **严重Bug**
- 位置：[src/components/ui/input/Input.vue](src/components/ui/input/Input.vue)
- 原因：组件缺少 `modelValue` prop 和 `update:modelValue` emit
- 影响：
  - ❌ `v-model` 双向绑定完全不工作
  - ❌ `searchKeyword.value` 始终为空字符串
  - ❌ **整个项目的Input组件都无法正确使用v-model**
  - ❌ 这是一个全局性的组件bug

**3. 用户体验问题：自动搜索频繁触发**
- 位置：[src/views/MusicList.vue:26](src/views/MusicList.vue#L26)
- 原因：`@input` 事件绑定了防抖搜索
- 影响：每次输入都等待500ms后自动搜索，用户无法控制时机

## 🎯 解决方案

### 实施的改动

**1. 修复Input组件v-model实现** ⭐ **核心修复**
**文件**: [src/components/ui/input/Input.vue](src/components/ui/input/Input.vue)

**改动内容**:
```vue
<template>
  <input
    :class="cn(...)"
    :value="modelValue"           <!-- ✅ 新增：绑定值 -->
    @input="onInput"              <!-- ✅ 新增：监听输入 -->
    v-bind="$attrs"
  />
</template>

<script setup>
import { cn } from '@/utils'

const props = defineProps({
  class: { type: String, default: '' },
  modelValue: {                   <!-- ✅ 新增：接收父组件值 -->
    type: [String, Number],
    default: ''
  }
})

const emit = defineEmits(['update:modelValue'])  <!-- ✅ 新增：定义事件 -->

function onInput(event) {          <!-- ✅ 新增：处理输入事件 -->
  emit('update:modelValue', event.target.value)
}
</script>
```

**影响范围**: 整个项目的Input组件现在都能正确使用v-model

## 📈 技术要点

### Vue 3 v-model 实现原理

**父组件使用**:
```vue
<Input v-model="searchKeyword" />
```

**编译为**:
```vue
<Input
  :model-value="searchKeyword"
  @update:model-value="newValue => searchKeyword = newValue"
/>
```

**子组件必须实现**:
```vue
<script setup>
// 1. 接收值
defineProps({
  modelValue: { type: [String, Number], default: '' }
})

// 2. 定义事件
defineEmits(['update:modelValue'])
</script>

<template>
  <!-- 3. 绑定值和事件 -->
  <input
    :value="modelValue"
    @input="$emit('update:modelValue', $event.target.value)"
  />
</template>
```
