### 步骤 1：创建音频格式处理工具
**文件**：`src/utils/audioFormat.js`
**功能**：多格式/多音质选择逻辑

**关键函数**：
- `canPlayFormat(format)`: 检测浏览器是否支持指定格式
- `selectBestAudioSource(sources)`: 选择最佳音频源（优先级：浏览器支持 > 高音质）
- `getFormatMimeType(format)`: 获取 MIME 类型

**逻辑概要**：
1. 使用 `audio.canPlayType()` 检测支持的格式
2. 过滤出浏览器支持的音频源
3. 按音质排序（high > medium > low）
4. 返回最佳匹配的 URL

**预期结果**：
```javascript
selectBestAudioSource([
  { format: 'mp3', quality: 'high', url: '...' },
  { format: 'ogg', quality: 'low', url: '...' }
])
// => { format: 'mp3', quality: 'high', url: '...' }
```

### 步骤 2：创建播放器 Store（核心状态管理）
**文件**：`src/stores/player.js`
**功能**：全局播放状态和播放列表管理

**状态定义**：
```javascript
{
  // 播放状态
  currentTrack: null,           // 当前播放歌曲对象
  isPlaying: false,             // 是否正在播放
  currentTime: 0,               // 当前播放时间（秒）
  duration: 0,                  // 总时长（秒）
  volume: 0.8,                  // 音量 (0-1)

  // 播放列表
  playlist: [],                 // 播放队列
  currentIndex: -1,             // 当前播放索引
  playMode: 'sequential',       // 播放模式: sequential/random/repeat-one

  // 音频源
  currentAudioSource: null,     // 当前选中的音频源对象
  loading: false,               // 音频加载状态
}
```

**关键方法**：
- `playTrack(track)`: 播放指定歌曲（添加到列表并播放）
- `addToPlaylist(track)`: 添加到播放列表
- `togglePlay()`: 切换播放/暂停
- `playNext()`: 播放下一曲
- `playPrevious()`: 播放上一曲
- `setPlayMode(mode)`: 设置播放模式
- `clearPlaylist()`: 清空播放列表
- `removeFromPlaylist(index)`: 从列表移除
- `updateTime(time)`: 更新播放时间
- `setVolume(volume)`: 设置音量
- `seek(time)`: 跳转到指定时间

**逻辑概要**：
1. 播放歌曲时自动选择最佳音频源（调用 audioFormat.js）
2. 根据播放模式计算下一曲索引（顺序/随机/循环）
3. 持久化音量设置到 localStorage
4. 自动处理播放列表去重（同一首歌只保留一次）

**预期结果**：全局可访问的播放器状态，任意组件可调用播放方法

### 🔧 实施过程中遇到的问题与解决方案

#### 问题 1：进度条跳转不生效

**问题描述**：
- 点击进度条跳转时，UI 显示的时间更新了，但音频播放位置没有改变
- Store 的 `seek()` 方法只更新了状态，没有实际控制 Audio 元素

**根本原因**：
```javascript
// ❌ 问题代码
function seek(time) {
  currentTime.value = time;  // 只更新了 Store 状态
  // 缺少：audio.currentTime = time
}
```

**解决方案**：
- 方案演进：
  1. **尝试 1**：依赖注入模式 - 在 useAudioPlayer 中注册 `seekAudio` 方法到 Store
     - 问题：时序问题，组件加载顺序不确定
  2. **最终方案**：让 PlayerProgress 组件直接使用 useAudioPlayer

```javascript
// ✅ 最终方案
// PlayerProgress.vue
const { seek: seekAudio } = useAudioPlayer()

function handleSeekEnd(event) {
  const newTime = parseFloat(event.target.value)
  playerStore.seek(newTime)    // 立即更新 UI
  seekAudio(newTime)           // 控制 Audio 元素
}
```

**设计原则**：
- ✅ 组件自给自足，不依赖外部初始化顺序
- ✅ UI 响应快（立即更新 Store）
- ✅ 播放流畅（直接控制 Audio）

---

#### 问题 2：播放卡顿现象

**问题描述**：
- 播放音乐时出现一卡一卡的现象
- 控制台无错误，但播放不流畅

**根本原因**：
- 最初的方案使用了 `watch` 监听 `Store.currentTime`，当时间变化 > 0.1 秒时同步到 Audio
- 但 `timeupdate` 事件频率不稳定（约 0.1 秒一次），可能误判为跳转操作
- 形成循环依赖：`timeupdate → updateTime → watch → 设置 audio.currentTime → timeupdate`

```javascript
// ❌ 问题代码（导致循环依赖）
watch(
  () => playerStore.currentTime,
  (newTime, oldTime) => {
    if (Math.abs(newTime - oldTime)  > 0.1) {
      audio.currentTime = newTime  // 误判，导致卡顿
    }
  }
)
```

**解决方案**：
- 移除 watch 监听器，避免循环依赖
- PlayerProgress 直接调用 useAudioPlayer 的 seek 方法
- 让 `timeupdate` 事件自然更新 Store 状态

**设计原则**：
- ✅ 避免循环依赖
- ✅ 职责分离：Store 管理状态，useAudioPlayer 控制 Audio
- ✅ 信任原生 API 的 `timeupdate` 事件

---

#### 问题 3：AbortError 错误

**问题描述**：
```
AbortError: The play() request was interrupted by a call to pause()
AbortError: The play() request was interrupted by a new load request
```

**根本原因**：
- **场景 1（pause 中断）**：快速切换播放/暂停时，`play()` Promise 还在执行就调用了 `pause()`
- **场景 2（load 中断）**：切换歌曲时，`audio.src = newUrl` 立即中断了旧的 `play()` 请求
- 这是 HTML5 Audio API 的**正常行为**，不是 bug

**解决方案**：

```javascript
// ✅ 优化后的 play() 方法
async function play() {
  const audio = getAudioInstance()
  // 避免重复播放
  if (!audio.paused) {
    playerStore.resume()
    return
  }

  try {
    await audio.play()
    playerStore.resume()
  } catch (error) {
    if (error.name === 'NotAllowedError') {
      console.warn('浏览器阻止自动播放，需要用户交互')
      playerStore.pause()
    } else if (error.name === 'AbortError') {
      // AbortError 是正常的切换行为，静默处理
      playerStore.pause()
    } else {
      console.error('播放失败:', error)
      playerStore.pause()
    }
  }
}
```

**错误级别划分**：
| 错误类型 | 级别 | 说明 |
|---------|------|------|
| `AbortError` | 静默处理 | 正常的切换行为 |
| `NotAllowedError` | `console.warn` | 浏览器自动播放策略限制 |
| 其他错误 | `console.error` | 真正需要关注的错误 |

**设计原则**：
- ✅ 防御性编程：检查 `audio.paused` 避免重复调用
- ✅ 错误分级：区分正常行为和异常情况
- ✅ 控制台干净：只在必要时打印错误

#### 问题 4：歌手信息显示"未知歌手"

**问题描述**：
- PlayerBar 显示"未知歌手"，但数据中包含 `artists` 数组
- 数据结构：
  ```javascript
  artists: [
    { id: 1, name: "周杰伦", role: "composer" },
    { id: 2, name: "周杰伦", role: "singer" }
  ]
  ```

**根本原因**：
- PlayerInfo 组件期望 `currentTrack.artistNames` 字段
- 但实际数据只有 `artists` 数组，需要筛选 `role === 'singer'` 的元素

**解决方案**：

```javascript
// ✅ 新增工具函数 extractArtistNames
function extractArtistNames(artists) {
  if (!artists || !Array.isArray(artists) || artists.length === 0) {
    return '未知歌手';
  }

  // 筛选出 role 为 singer 的艺术家
  const singers = artists.filter((artist) => artist.role === 'singer');

  if (singers.length === 0) {
    // 降级：使用第一个艺术家
    return artists[0]?.name || '未知歌手';
  }

  // 多个歌手用 / 分隔
  return singers.map((singer) => singer.name).filter(Boolean).join(' / ');
}

// 在 playTrack/playNext/playPrevious 中使用
const processedTrack = {
  ...track,
  artistNames: extractArtistNames(track.artists)
};
currentTrack.value = processedTrack;
```

**显示效果**：
- 单个歌手：`周杰伦`
- 多个歌手：`周杰伦 / 陈奕迅`
- 无歌手：`未知歌手`

**设计原则**：
- ✅ 数据转换集中处理
- ✅ 优雅降级：无 singer 时使用第一个艺术家
- ✅ DRY：三个播放方法复用同一逻辑
- ✅ 空值保护：`filter(Boolean)` 过滤空名称

### 🎯 关键技术决策

#### 1. Store 与 Composable 的职责划分

| 模块 | 职责 |
|------|------|
| **Store (player.js)** | 状态管理（播放状态、播放列表、音频源） |
| **Composable (useAudioPlayer.js)** | Audio 元素控制（play/pause/seek/volume） |
| **组件** | UI 交互，调用 Store 和 Composable |

**原因**：
- Store 不应直接操作 Audio 元素（违反单一职责原则）
- Composable 可以在多个组件中复用
- 清晰的职责划分便于测试和维护

#### 2. 为什么不用依赖注入

最初尝试在 useAudioPlayer 中注册 `seekAudio` 方法到 Store，但发现：
- 组件加载顺序不确定，可能导致方法未注册
- 增加了模块间的耦合
- 调试困难（不确定方法何时注册）

**最终选择**：PlayerProgress 直接使用 useAudioPlayer
- 组件自给自足
- 不依赖外部初始化顺序
- 代码更清晰

#### 3. 为什么不手动更新 currentTime

在 `seek()` 方法中移除了 `playerStore.updateTime(audio.currentTime)`：
```javascript
// ❌ 不推荐
function seek(time) {
  audio.currentTime = time
  playerStore.updateTime(audio.currentTime)  // 可能导致循环
}

// ✅ 推荐
function seek(time) {
  audio.currentTime = time
  // 让 timeupdate 事件自动更新
}
```

**原因**：
- 信任原生 API 的 `timeupdate` 事件
- 避免状态冲突
- 减少代码复杂度
