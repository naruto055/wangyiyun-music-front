# 视频接口三段式适配

## 任务背景

- 后端视频能力已从旧版“`POST /api/video/parse` 同步返回音频地址”切换为三段式调用。
- 当前前端仍有旧协议依赖，导致视频解析页与 B 站搜索页播放链路存在兼容问题。
- 本次范围不包含 B 站搜索页“保存为音乐”能力。

## 选定方案

- 方案：最小侵入式适配
- 原则：
  - KISS：保持现有页面结构与状态组织
  - YAGNI：不额外引入新的 store / composable
  - DRY：复用现有播放器轨道结构与统一请求处理

## 执行步骤

1. 更新 `src/api/video.js`
   - 保留 `parseVideo`
   - 新增 `prepareAudio`
   - 新增 `downloadVideo`

2. 调整 `src/utils/trackAdapter.js`
   - 将 `adaptVideoToTrack` 输入语义切换为“已准备好的音频结果”
   - 保持播放器所需的统一音轨结构

3. 重构 `src/views/VideoParser.vue`
   - `parseVideo` 仅获取元数据
   - 播放音频时调用 `prepareAudio`
   - 下载音频时调用 `prepareAudio`
   - 下载视频时调用 `downloadVideo`
   - 按 `availableActions`、`audioSupported`、`videoSupported` 动态控制按钮

4. 重构 `src/views/BilibiliSearch.vue`
   - 播放链路改为 `parse -> prepareAudio -> play`
   - 保持“保存为音乐”为未实现状态

5. 清理旧协议残留
   - 删除不再需要的导入、注释和错误假设

6. 进行本地验证
   - 优先执行轻量级构建或语法检查

## 预期结果

- `VideoParser` 完整适配新版三段式协议
- `BilibiliSearch` 的播放链路恢复可用
- 前端不再依赖 `parse` 返回 `audioUrl`
- UI 动作展示与后端能力字段保持一致
