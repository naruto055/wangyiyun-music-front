**日期：** 2026-02-05
**问题类型：** Bug 修复
**优先级：** 高
**状态：** ✅ 已解决

## 📋 问题概述

### 问题 1：后端数据结构不一致问题

**问题描述：**
后端视频解析接口返回的数据与音乐详情接口返回的数据结构不一致，导致前端播放器无法统一处理两种数据源。

**影响范围：**
- 视频解析后的音频无法播放
- 音乐详情播放可能受影响
- 播放器状态管理混乱

### 问题 2：音频源字段名错误

**问题描述：**
点击视频解析后的"播放"按钮，音频无法播放，控制台输出：
```
没有浏览器支持的音频格式，使用降级方案
```

**影响范围：**
- 视频解析功能完全无法使用
- 用户体验严重受损

6. 打印"**没有可用的音频源**" → 无法播放

## 💡 解决方案

### 方案选择：前端适配器模式 ✅

**原因：**
- ✅ 职责分明 - 后端保持各自服务边界，前端统一处理
- ✅ 扩展性强 - 支持将来添加更多平台（YouTube、抖音）
- ✅ 用户体验 - 清晰标识临时文件，避免混淆
- ✅ 维护性好 - 集中管理数据转换逻辑

**替代方案（未采用）：**
- ❌ 后端统一接口格式 - 语义不清晰，视频解析无法提供完整音乐信息
- ❌ 前端多个播放入口 - 代码重复，维护成本高

```javascript
/**
 * 将视频解析结果转换为音轨对象
 */
export function adaptVideoToTrack(videoParseResult) {
  return {
    // 基础信息
    id: `video_${videoParseResult.sourceVideoId}`,
    title: videoParseResult.title,
    coverUrl: videoParseResult.coverUrl,
    duration: videoParseResult.duration,

    // 音频源（关键修复：使用 url 而非 sourceUrl）
    audioSources: [
      {
        url: videoParseResult.audioUrl,        // ✅ 正确字段名
        format: videoParseResult.audioFormat,
        quality: 'high',
      },
    ],

    // 缺失字段 - 使用默认值
    artistNames: '视频音频',
    fileUrl: null,
    lyrics: null,
    playCount: 0,
    favoriteCount: 0,
    releaseDate: null,
    albumName: '视频提取',
    categoryName: '其他',
    artists: [],
    tags: ['视频音频'],

    // 元数据（区分数据来源）
    _source: 'video_parse',
    _platform: videoParseResult.platform,
    _sourceVideoId: videoParseResult.sourceVideoId,
    _expiresAt: videoParseResult.expiresAt,
    _isTemporary: true,
  }
}

/**
 * 将音乐详情转换为音轨对象
 */
export function adaptMusicToTrack(musicDetail) {
  const artistNames = musicDetail.artists
    ?.filter((a) => a.role === 'singer')
    .map((a) => a.name)
    .join('/') || '未知歌手'

  return {
    // 基础信息
    id: musicDetail.id,
    title: musicDetail.title,
    coverUrl: musicDetail.coverUrl,
    duration: musicDetail.duration,

    // 音频源（关键修复：保留 music store 已构造的 audioSources）
    audioSources: musicDetail.audioSources || null,  // ✅ 不覆盖为 null

    // 完整字段
    artistNames,
    fileUrl: musicDetail.fileUrl,
    lyrics: musicDetail.lyrics,
    playCount: musicDetail.playCount,
    favoriteCount: musicDetail.favoriteCount,
    releaseDate: musicDetail.releaseDate,
    albumName: musicDetail.albumName,
    categoryName: musicDetail.categoryName,
    artists: musicDetail.artists,
    tags: musicDetail.tags || [],

    // 元数据
    _source: 'music_detail',
    _isTemporary: false,
  }
}

/**
 * 辅助工具函数
 */
export function isTemporaryTrack(track) {
  return track._isTemporary === true || track._source === 'video_parse'
}

export function getExpiryHint(track) {
  if (!isTemporaryTrack(track) || !track._expiresAt) return null

  const expiresAt = new Date(track._expiresAt)
  const now = new Date()
  const remainingMs = expiresAt - now

  if (remainingMs <= 0) return '音频已过期'

  const remainingMinutes = Math.floor(remainingMs / 1000 / 60)
  if (remainingMinutes < 60) return `${remainingMinutes} 分钟后过期`

  return '1 小时后过期'
}

export function isTrackExpired(track) {
  if (!isTemporaryTrack(track) || !track._expiresAt) return false
  return new Date() >= new Date(track._expiresAt)
}
```

## 📋 标准音频源格式

播放器期望的标准格式（基于 `src/utils/audioFormat.js`）：

```javascript
{
  url: 'http://localhost:8910/audio/xxx.mp3',  // 必须是 url 字段
  format: 'mp3',                                // 音频格式（mp3/flac/m4a/ogg/wav）
  quality: 'high',                              // 音质等级（high/medium/standard/low）
}
```

**格式检查流程：**
1. `selectBestAudioSource` 检查 `!source.format || !source.url` → 过滤无效源
2. `canPlayFormat` 检查浏览器支持的格式 → 过滤不支持的格式
3. 按 `quality` 排序 → 选择最高音质

**注意事项：**
- ❌ 不要使用 `sourceUrl`、`audioUrl`、`src` 等其他字段名
- ❌ 不要添加不必要的字段（如 `id`）
- ✅ `format` 必须是小写（`mp3` 而非 `MP3`）
- ✅ `quality` 可选值：`high`、`medium`、`standard`、`low`
