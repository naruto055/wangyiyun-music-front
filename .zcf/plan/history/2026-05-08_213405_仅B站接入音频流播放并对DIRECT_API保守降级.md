# 仅B站接入音频流播放并对DIRECT_API保守降级

> 创建时间：2026-05-07 23:56:01
> 当前模式：执行

## 背景

后端新增 `POST /api/video/audio/stream`，用于“立即播放”场景直接获取音频流地址。当前仅 B 站支持直连解析器，前端播放器仍基于原生 `HTMLAudioElement`，无法可靠附带自定义请求头，因此需要在接入新接口时对 `DIRECT_API + headers` 采用保守降级策略。

## 目标

- 仅对 `BILIBILI` 启用新的快速播放链路
- 保留 `parse` 作为元数据展示入口
- 保留 `prepareAudio` 作为下载音频和播放降级链路
- 统一三个播放入口的行为

## 保守降级策略

- `sourceType = YTDLP_FALLBACK`：直接接入当前播放器
- `sourceType = DIRECT_API` 且 `headers` 为空：尝试直接接入当前播放器
- `sourceType = DIRECT_API` 且 `headers` 非空：降级为调用 `prepareAudio` 获取临时音频资源后播放

## 风险说明

- 当前浏览器播放器不支持可靠附带 `Referer` / `User-Agent`
- 非 B 站平台不启用新播放链路
- 若后端未来扩展更多平台，前端可复用本次抽象继续扩展
