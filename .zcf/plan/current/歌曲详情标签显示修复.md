**影响范围**：
- 文件：`src/components/MusicDetailDialog.vue`
- 影响行数：标签渲染部分（第38-40行）

## 根本原因

后端接口实现与 API 文档不一致：
- **文档描述**：`tags` 为对象数组，包含 `id` 和 `name` 字段
- **实际返回**：`tags` 为字符串数组，直接包含标签名称

## 执行步骤

### 步骤 1：修改标签渲染逻辑

**文件**：`src/components/MusicDetailDialog.vue`

**位置**：第38行

**修改前**：
```vue
<span v-for="tag in musicDetail.tags" :key="tag.id" class="rounded-full border border-border px-3 py-1 text-xs">
	{{ tag.name }}
</span>
```

**修改后**：
```vue
<span v-for="(tag, index) in musicDetail.tags" :key="index" class="rounded-full border border-border px-3 py-1 text-xs">
	{{ tag }}
</span>
```

**说明**：
- 移除 `tag.id`，使用数组索引 `index` 作为 key
- 移除 `tag.name`，直接显示 `tag`（因为 tag 本身就是字符串）

**预期结果**：标签能正确显示文字信息

## 技术要点

### Vue 列表渲染最佳实践

**数组索引作为 key**：
- 适用于静态列表（不会重新排序、过滤）
- 标签数据通常满足此条件
- 性能影响可忽略

**避免使用对象属性作为 key**：
- 当数组元素是简单类型（字符串、数字）时，使用索引更安全
- 避免访问不存在的属性导致的错误
